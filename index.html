「Webコンテンツ作成」の成果

本テーマについて

Webページの基本的な記述方法、HTML, CSS, JavaScriptなどを授業毎に使用し、 その操作を慣れるとともに、最終的に学んだことの総復習として、自身で簡単な 検索ページの作成を目指すものである。

プロジェクト課題について

私はプロジェクト課題の題材として、「グルメ情報」について取り扱いました。主な機能としては、検索欄、検索ボタン、検索結果 の合計件数の表示などがあり、店のジャンルを入力することで、情報が表示されるようになっています。また、その他の機能で、 トップページに戻るボタンなどは自身で追加しました。このプロジェクト課題では、特に、ネットから取り出したデータをどう表示 させるか、再度検索したとき、先ほどの検索結果をどう削除するのかなどの問題があり、それらは、コンソールの出力結果でどう構成 されているのかを見たり、while文を使って削除するなどをして解決しました。 (200文字以上)

プロジェクト課題で作成したページ: project.html

数当てゲーム

JavaScript、HTMLを利用し、ユーザーが入力した数値と、ランダム生成された数値が合っているのかを当てるゲーム。 最初の方では、ユーザーの予想を固定し、コンソールにて出力結果を出したが、その後の授業で、結果テキストを更新 することや、ユーザーの入力を反映することなどを学んだ。 機能としては、予想テキストボックスに1~10の整数を入力してもらい、表示ボタンを押した後、何回目の予想なのか。 答えと自身の予想が合っているのか、もし違っていたら予想が大きいのか、小さいのかが表示される。また、一定の 回答回数を超えた場合に限り、ゲームが終了し、答えが表示される。その後に回答をしても、「すでにゲームは終わっています」 と表示されるようになっている。 (200文字以上)

各授業回の学習事項

第1回 WWWとHTML

Webページの基本的な記述方法と、HTMLを学ぶ回。内容としては、基本的なheadやbodyの構文から、 その中の<p>やその他、記号(<など)、協調、文字のリスト化、ハイパーリンク、画像の 貼り付け等を作成、学ぶ回だった。

演習の成果:演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

Webページにおけるデザインやレイアウトを行うために使われるCSSの基本的な使い方を学ぶ回だった。 今回では、特に配置や図形などがよく利用されており、実際に使ってみて自身のWebサイトが見やすく なった反面、調節するのは少し工夫が必要、また、慣れてないためか時間もかかり大変だと感じた。

演習の成果:演習2-3: CSSの作成

第3回 JavaScript の基本

Webページにおけるプログラミング部分のJavaScriptについて学ぶ回だった。今回はJavaScriptを使っての システムの構築の仕方について、基本的な機能や知識を学ぶ回であり、実際にWebページに実装はできていないが、 裏で必要な情報を出すシステムを動かせる程度の知識が備わった。

演習の成果:演習3-6: JavaScriptの作成

第4回 DOM と JavaScript

WebページにおけるJavaScriptの機能で、前回はコンソール上での出力を行ったが、それをコンソール上ではなく、 html上に情報を適宜追加、削除を学ぶ回だった。これは、Javascriptでプログラムすることで、 少ない作業量で、より多くの情報を扱えるような技術であり、その情報の削除、追加の方法を学べた。

演習の成果:演習4-2~5: DOMプログラミング演習

第5回 イベント処理

WebページにおけるJavaScriptの機能、イベントについて。ボタン、テキストボックスなどの設置、それを押したときなどにイベントを どう起こすのかを、JavaScriptでプログラムし、実際にサイトに反映させることを行った。この機能により、検索することができるのは もちろんのこと、よりWebサイトに動きを持たせることが可能になった。

演習の成果:演習5-3: イベントの種類

工夫点、苦心点、強調点、感想

今回のWebコンテンツ作成では、新たに3つのプログラム言語、JavaScript、HTML、CSSを学ぶことができました。JavaScriptに関しては、 Webページにおける動作的な部分を担当しており、特にユーザーインタラクションや動的なコンテンツの処理において重要な役割でした。 Javaという言語と名前は似ていますが、全く異なる言語であり、その違いを理解するのには時間がかかりました。特に、DOM操作（Document Object Model） によるHTML要素の追加や削除などのプログラムを書くことは最初は混乱し、どこから手をつければよいのかわからずに苦労しました。これを克服するために、 授業内容を何度も見返し、実際にコードを書きながら試行錯誤を繰り返しました。 HTMLに関しては、他の2つの言語と比べてもシンプルな構造を持っており、タグを使用して要素を定義するだけで基本的なページを構築できるため、 比較的容易に習得することができました。しかし、CSSはその簡潔さとは対照的に、Webサイト全体のデザインを担当するため、学ぶことが非常に多かったです。 CSSプロパティの種類は多岐にわたり、どれを使えばどのようなデザインが実現できるのかを理解するのには時間がかかりました。 授業時間が限られていたため、CSSについて深く学ぶ機会が少なく、最終的に作成されたWebコンテンツはシンプルなデザインとなってしまいました。デザインに関しては、 より高度な技術を駆使して、視覚的に魅力的なページを作り上げることができなかったため、物足りなさを感じることが多々ありました。具体的には、このページ内のテキストを 中央に配置して読みやすくしたり、右下にトップページに戻るボタンを実装するなどの基本的なデザイン変更は行いましたが、これらはほんの一部に過ぎませんでした。 グルメ情報のページにおいても、トップページへ戻るボタンや検索欄を中央に配置し、その横に検索ボタンを置くなど、基本的な機能を追加しましたが、これらはすべて自身が 調べながらの作業でした。完成したページを見たときに、多少は利便性が上がったと感じましたが、視覚的に見やすいページにできたかどうかについては疑問が残りました。 今後の課題として、より魅力的で使いやすいWebページを作成するために、CSSのさらなる学習と実践が必要であると感じました。特に、色彩理論やタイポグラフィ、 アニメーションの実装など、より高度なデザイン技術を習得し、それを実際のプロジェクトに反映させることが求められます。 このように、今回のWebコンテンツ作成を通じて、プログラム言語の習得とその応用に関して多くのことを学びました。特に、JavaScript、HTML、CSSという3つの言語が どのように連携して動作し、どのようにして魅力的なWebページを構築できるのかを理解することができました。この経験を通じて得た知識とスキルを今後のプロジェクトに活かし、 さらに高度なWebコンテンツを作成できるよう努力していきたいと思います。
